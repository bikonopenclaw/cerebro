# Identidade visual Bikon para relatórios

Objetivo: centralizar os arquivos de marca da Bikon para o Kowalski usar em relatórios técnicos e apresentações para clientes.

## Pastas
- `logos/`: logos oficiais, versões horizontal, vertical, positiva, negativa, ícone.
- `cores/`: paleta oficial, códigos HEX/RGB/CMYK se disponíveis.
- `fontes/`: fontes da marca ou indicação de fontes equivalentes.
- `referencias/`: materiais existentes, apresentações, PDFs, posts, timbrados.
- `rascunhos/`: propostas criativas de layout feitas pelo Kowalski.
- `modelos-aprovados/`: templates finais aprovados para uso recorrente.

## Regra de uso
Todo relatório, documento ou PDF gerado pelo Kowalski deve usar a identidade visual da Bikon, inclusive testes, rascunhos e validações internas.

Kowalski pode ser criativo e sugerir modelos de apresentação de relatórios, mas nunca deve entregar material com cara genérica de ferramenta.

Depois que Hebert/Puppet Master aprovar um modelo, ele deve ser salvo em `modelos-aprovados/` e passar a ser usado como padrão para manter consistência profissional.

Entrega sem identidade visual Bikon conta como incompleta e deve ser refeita antes de ser enviada.

## Direção inicial
- Relatórios limpos, executivos e fáceis de ler.
- Visual Bikon, sem cara de relatório genérico de ferramenta.
- Priorizar clareza: status, riscos, ações recomendadas e evidências.
- Usar português BR direto, profissional e sem floreio.
- Nunca usar travessão em copy pública.
