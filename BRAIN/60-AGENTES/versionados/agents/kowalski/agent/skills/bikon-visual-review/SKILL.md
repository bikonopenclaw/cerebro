# Bikon Visual Review

Use esta skill quando Robotnik, Kowalski ou Puppet Master forem revisar ou preparar uma peça visual pública ou semi-pública da Bikon: carrossel, post, relatório, PDF, apresentação, landing page, proposta visual ou material comercial.

A skill é auxiliar. Ela não substitui estratégia, copy, identidade visual aprovada ou decisão do Hebert.

## Objetivo

Garantir que a peça pareça Bikon: técnica, confiável, direta e operacional. A Bikon vende continuidade, infraestrutura, SNOC, backup em nuvem e redes corporativas. Não vender estética hacker, startup roxa, cyberpunk ou medo barato.

## Quando Usar

Use em:
- Carrossel do Instagram.
- Arte de post estático.
- Relatório técnico para cliente.
- PDF executivo.
- Proposta comercial com visual Bikon.
- Landing page ou seção de site.
- Peça criada por Robotnik que precisa de validação visual.
- Documento criado por Kowalski que precisa manter padrão Bikon.

Não use para:
- Texto puro sem arte.
- Diagnóstico técnico sem entrega visual.
- Código backend.
- Comunicação externa urgente onde o conteúdo importa mais que o acabamento.

## Posição no Fluxo

1. Robotnik define pauta, copy, legenda e roteiro visual.
2. Kowalski garante padrão Bikon quando houver documento, relatório, PDF ou apresentação.
3. Bikon Visual Review revisa a peça antes de enviar para Hebert.
4. Puppet Master decide se está pronto para validação.
5. Hebert aprova, pede ajuste ou reprova.

Publicação real continua bloqueada sem aprovação explícita do Hebert.

## Identidade Visual Operacional

A Bikon deve parecer:
- Confiável.
- Técnica.
- Clara.
- Sóbria.
- Executiva.
- Próxima de PME brasileira.
- Orientada a risco operacional e continuidade.

A Bikon não deve parecer:
- Hacker/cyberpunk.
- Matrix.
- SaaS genérico roxo/azul neon.
- Agência criativa tentando impressionar.
- Terror de segurança barato.
- Material de fabricante.
- Canva genérico.

## Paleta Recomendada

Base escura:
- Fundo principal: `#101820`
- Fundo alternativo: `#0B3D5C`
- Bloco técnico: `#1F2933`
- Texto principal: `#F8FAFC`
- Texto secundário: `#94A3B8`
- Linha/borda discreta: `#1E5668`

Base clara:
- Fundo claro: `#F8FAFC`
- Superfície: `#FFFFFF`
- Texto principal: `#020617`
- Texto secundário: `#64748B`
- Borda: `#E2E8F0`

Acentos:
- Ciano Bikon: `#00A8B3`
- Ciano forte: `#1EE3E8`
- Azul confiança: `#0369A1`
- Vermelho alerta: `#DC2626`, usar pouco e só para risco real.

Regra: ciano destaca verbo, risco, ação, status ou evidência. Ciano não vira tema inteiro.

## Tipografia

Preferência:
- Inter, Arial, Helvetica ou Plus Jakarta Sans.
- Título com peso 700 ou 800.
- Corpo com peso 400 ou 500.
- Evitar fonte monoespaçada como corpo principal.
- Evitar fonte futurista, decorativa ou pesada demais.

Regra para posts:
- Título curto, forte e legível no celular.
- Uma ideia por slide.
- Corpo mínimo.
- Linha curta.
- Não reduzir fonte para caber texto demais.

## Grid para Carrossel

Formato base: 1080x1080.

Regras:
- Margem segura mínima: 80 px.
- Logo com respiro, nunca espremido.
- Título no topo ou centro, sem encostar em borda.
- Máximo 3 bullets por slide.
- Um bloco visual principal por slide.
- CTA sempre no último slide.
- Evitar rodapé poluído.
- Evitar texto em cima de fundo sem contraste.

Estrutura recomendada para post técnico:
1. Gancho direto.
2. Contexto operacional.
3. Impacto para PME.
4. Evidência ou referência.
5. Virada prática.
6. Ação/CTA.

## Grid para Relatórios e PDFs

Regras:
- Capa limpa com título, cliente, data e Bikon.
- Sumário executivo antes do detalhe técnico.
- Organizar por risco, impacto e ação.
- Tabelas simples, com zebra discreta se necessário.
- Ciano só para status, recomendação ou destaque real.
- Evitar card dentro de card.
- Evitar decoração sem função.

## Landing Page

Direção recomendada:
- Trust & Authority.
- Minimal & Direct.
- Swiss Modernism 2.0.
- Real-Time / Operations quando houver painel, SNOC ou métrica.

Estrutura recomendada:
1. Hero direto com promessa clara.
2. Dor operacional concreta.
3. Como a Bikon resolve.
4. Prova, processo ou indicadores.
5. CTA: solicitar checagem técnica.

Evitar:
- Hero genérico com gradiente roxo.
- Imagem stock escura e sem relação.
- Frase bonita sem substância.
- Promessa de segurança total.

## Checklist de Revisão

Antes de enviar para Hebert, responder:

1. Parece Bikon ou parece template genérico?
2. O texto principal é legível em celular em 3 segundos?
3. Tem uma ideia por slide/página/bloco?
4. O logo respira?
5. O contraste passa no olho sem esforço?
6. O ciano está destacando algo útil ou virou enfeite?
7. Tem estética hacker/Matrix/cyberpunk? Se sim, refazer.
8. Tem medo barato? Se sim, cortar.
9. O CTA é claro e sóbrio?
10. Dá para um empresário PME entender sem saber jargão técnico?

## Anti-Padrões Bloqueados

Bloquear ou pedir refação quando aparecer:
- Cyberpunk, Matrix, hacker hooded figure.
- Cadeado, caveira, alerta vermelho em excesso.
- Gradiente roxo/azul dominante.
- Muito texto no slide.
- Fonte menor para compensar copy longa.
- Logo espremido.
- Card dentro de card.
- Foto stock sem conexão operacional.
- CTA exagerado.
- Promessa absoluta, como 'segurança garantida'.

## Saída Esperada

Ao revisar, entregar no máximo:

- Veredito: aprovado, ajustar ou reprovar.
- 3 ajustes prioritários.
- Risco visual principal.
- Se for carrossel: estrutura slide a slide revisada.
- Se for relatório/landing: checklist de correção.

Formato curto. Não virar aula de design.

## Exemplo de Direção Correta

Tema: firewall antigo.

Capa:
Firewall antigo não é economia.
É risco.

Slides:
1. Gancho.
2. Borda da rede como porta de entrada.
3. Sem suporte, menos atualização e controle.
4. Evidência externa sem copiar regra estrangeira cegamente.
5. O problema é não saber o que está exposto.
6. CTA: solicite checagem técnica da rede.

Tom: sério, direto, sem pânico.

## Regra Final

A skill serve para aumentar consistência visual e reduzir erro de gosto. Se ela entrar em conflito com identidade Bikon, decisão do Hebert ou revisão do Kowalski, ela perde.
