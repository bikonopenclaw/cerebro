# Modelo padrão diário ARX Backup

Este é o layout aprovado pelo Hebert em 2026-06-16 para relatórios diários da skill ARX Backup.

## Arquivos

- PDF aprovado: `modelo-padrao-relatorio-diario-arx-backup.pdf`
- HTML base editável: `modelo-padrao-relatorio-diario-arx-backup.html`

## Regras do modelo diário

- Usar identidade visual própria do ARX Backup.
- Período: últimas 24 horas.
- Não usar marca, logo, rodapé, assinatura, e-mail ou timbrado de terceiros.
- Contato padrão: `backup@arxcore.com.br`.
- Não citar Cove em documento para cliente.
- Não incluir seção de seleção protegida se a API não trouxer caminhos literais das pastas/arquivos.
- Só incluir seleção de backup quando `_SelectionFS`, `_SimpSelectionFS` ou outra fonte confiável trouxer caminhos literais preenchidos.
- Status, cor da faixa e indicadores precisam sempre vir dos dados reais da API, nunca do HTML base.
- Marcador aprovado: `ARX_APPROVED_DAILY_MODEL:modelo-padrao-relatorio-diario-arx-backup`.
