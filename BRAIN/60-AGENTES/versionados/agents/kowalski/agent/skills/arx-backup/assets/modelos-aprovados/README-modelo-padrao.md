# Modelo padrão ARX Backup

Este é o layout aprovado pelo Hebert em 2026-06-15 para relatórios mensais da skill ARX Backup.

## Arquivos

- PDF aprovado: `modelo-padrao-relatorio-mensal-arx-backup.pdf`
- HTML base editável: `modelo-padrao-relatorio-mensal-arx-backup.html`

## Regras do modelo

- Usar identidade visual própria do ARX Backup.
- Não usar marca, logo, rodapé, assinatura, e-mail ou timbrado de terceiros.
- Contato padrão: `backup@arxcore.com.br`.
- Documento executivo em 1 página sempre que possível.
- Horários em GMT-3.
- Não citar Cove em documento para cliente.
- Não incluir seção de recomendações. Usar `Pontos de acompanhamento`.
- Manter linguagem de report/acompanhamento operacional.
- O modelo é referência visual, não verdade operacional.
- Ao gerar relatório real, substituir status, cor da faixa e histórico dos últimos 28 dias pelos dados da API.
- Proibido manter faixa verde ou 28 segmentos verdes quando `T0`, `T7` ou `TB` indicarem falha, erro ou alerta.
